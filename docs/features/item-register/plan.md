# Plan. 옷 등록 (item-register)

## 단계 구성

| Phase | 이름                       | 목표                                                                         |
| ----- | -------------------------- | ---------------------------------------------------------------------------- |
| P1    | 백엔드 단건 등록 골격      | 사전서명 URL 발급·Item 메타 저장이 가능해진다. 분류는 stub.                  |
| P2    | AI 분류 v1 통합            | `POST /classify` 마이크로서비스 + Top-1/색상이 Item에 들어간다.              |
| P3    | 백엔드 일괄 등록           | batchId 발급·진행률 조회·재시도 큐가 동작한다.                               |
| P4    | 프론트 단건 등록 흐름      | 카메라/갤러리 진입·미리보기·압축·업로드·결과 화면이 완성된다.                |
| P5    | 프론트 일괄 등록 + 수정    | 멀티 선택·진행률 표시·카테고리 1탭 수정이 완성된다.                          |
| P6    | 검증·게이트                | T1(AI 정확도)·T4(등록 시간) 게이트 통과, 실기기 e2e·회귀 셋 측정 보고.       |

## 아키텍처 다이어그램

```
[Flutter]                                [NestJS API]                 [Classifier μSvc]      [R2/MinIO]
  │  1. POST /items/presign                  │                              │                    │
  ├─────────────────────────────────────────▶│                              │                    │
  │                                          │  StorageProvider.presign     │                    │
  │                                          ├──────────────────────────────┼───────────────────▶│
  │  { uploadUrl, objectKey }                │◀─────────────────────────────┼────────────────────┤
  │◀─────────────────────────────────────────│                              │                    │
  │  2. PUT uploadUrl (사진 bytes)            │                              │                    │
  ├─────────────────────────────────────────────────────────────────────────────────────────────▶│
  │                                          │                              │                    │
  │  3. POST /items { objectKey }            │                              │                    │
  ├─────────────────────────────────────────▶│                              │                    │
  │                                          │  4. POST /classify {key}     │                    │
  │                                          ├─────────────────────────────▶│                    │
  │                                          │                              │ 5. fetch from R2   │
  │                                          │                              ├───────────────────▶│
  │                                          │                              │◀───────────────────┤
  │                                          │  { top3, colorHex }          │                    │
  │                                          │◀─────────────────────────────│                    │
  │                                          │  6. INSERT Item              │                    │
  │  { id, category, photoUrl, ai... }       │                              │                    │
  │◀─────────────────────────────────────────│                              │                    │
  │                                                                                              │
  │  (batch flow: POST /items/batch → 항목별 1·2 반복 → GET /items/batch/:id 진행률 폴링)         │
```

## 테스트 매트릭스

| #   | 케이스                          | 입력                                            | 기대 결과                                                              |
| --- | ------------------------------- | ----------------------------------------------- | ---------------------------------------------------------------------- |
| 1   | 정상 단건 등록                  | 카메라 촬영 → 저장                              | 8초 내 옷장 그리드에 노출, category·colorHex·aiConfidence 채워짐       |
| 2   | 정상 일괄 등록                  | 갤러리 10장 선택                                | 진행률 0~100% 정상, 모두 옷장에 들어감, 토스트 표시                    |
| 3   | 카메라 권한 거부                | "허용 안 함" 선택                               | 권한 안내 화면 + 설정 진입 버튼, 등록 흐름 중단                        |
| 4   | 분류 타임아웃                   | /classify가 5초 초과                            | Item.category=null로 즉시 저장, 백그라운드 재분류 enqueue, UI 미분류   |
| 5   | 분류 신뢰도 < 0.5               | top1 confidence 0.4                             | category=null, "미분류" 섹션, 사용자에게 직접 선택 요청                |
| 6   | 사전서명 만료 후 PUT            | 1시간 후 업로드 시도                            | 클라이언트 1회 재발급·재시도, 2회 실패 시 실패 큐                      |
| 7   | 20장 초과 선택                  | 갤러리 21장 선택                                | 클라이언트 차단 토스트("최대 20장"), 배치 미생성                       |
| 8   | 배치 진행 중 새 배치 요청       | 진행 중 추가 `POST /items/batch`                | 409 Conflict, 기존 batchId 안내                                        |
| 9   | 네트워크 단절 후 복귀           | 비행기 모드 → 해제                              | 큐가 자동 재시도, 모두 성공 시 토스트                                  |
| 10  | 다른 사용자 Item 수정 시도      | 타 userId 토큰으로 `PATCH /items/:id/category`  | 403 Forbidden                                                          |
| 11  | 동일 batchId 중복 호출(멱등성)  | 같은 idempotency-key                            | 첫 응답과 동일 결과 반환, 중복 작업 미생성                             |
| 12  | T1 회귀 셋 측정                 | 라벨링된 100장                                  | Top-1 ≥80%, Top-3 ≥95%. 미달 시 게이트 차단                            |
| 13  | T4 등록 시간 측정               | 실기기 iOS·Android 저/고사양 각 10회            | 평균 ≤8초, P95 ≤12초                                                   |
| 14  | /classify 부하                  | 동시 1000 요청, 100장 회귀 셋 반복              | P95 ≤2초, 에러율 ≤0.5%                                                 |
| 15  | 보정 이벤트 기록                | Top-1 수정                                      | analytics에 `classify_corrected` 이벤트 1건, 원래/수정 카테고리 포함   |
