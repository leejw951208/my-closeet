## 아키텍처 다이어그램

```
[Tap "+" in Closet]
        │
        ▼
  Permission gate ── denied ──▶ [Permission help screen]
        │
        ▼
  [Camera/Gallery sheet]
        │
        ├── camera ──▶ [Camera capture]
        └── gallery ──▶ [Picker(single|multi)]
                            │
                            ▼
                    image compress (1600px, q85)
                            │
                            ▼
                    [Preview/Confirm]
                            │
                            ▼
                    ItemApi.presign → PUT → createItem
                            │
                            ▼
                    [Result screen] ── category? ──▶ auto-modal if null
                            │
                            ▼
                    옷장으로 복귀

(Batch path uses BatchApi.create → background queue → 2s poll → header badge → toast)
```

## 테스트 매트릭스

| #   | 케이스                          | 입력                              | 기대 결과                                                     |
| --- | ------------------------------- | --------------------------------- | ------------------------------------------------------------- |
| 1   | 정상 단건 등록                  | 카메라 촬영                       | 결과 화면에 Item·category 노출, 옷장에 추가                   |
| 2   | 카메라 권한 거부                | 권한 거부                         | 권한 안내 + 설정 진입 버튼                                    |
| 3   | 이미지 압축 실패                | 손상된 이미지                     | 에러 토스트 + 재촬영 안내                                     |
| 4   | 업로드 실패 + 자동 재시도        | PUT 5xx 2회                       | 2회 재시도 후 성공 또는 실패 화면                             |
| 5   | 정상 batch 10장                  | 갤러리 10장 선택                  | 백그라운드 진행률 0→100%, 완료 토스트, 모두 옷장에 들어감     |
| 6   | 21장 초과 선택                  | 갤러리 21장                       | 토스트 + 차단, batch 미생성                                   |
| 7   | 백그라운드 진행 중 화면 이동    | 진행 중 옷장 화면 진입            | 헤더 진행 배지 유지, 폴링 지속                                |
| 8   | 일부 항목 실패                  | 3개 중 1개 PUT 실패               | 실패 목록 화면, 재시도 버튼                                    |
| 9   | category=null 자동 모달          | 분류 신뢰도 < 0.5 결과            | 결과 화면 진입 즉시 카테고리 모달 자동 노출                   |
| 10  | 카테고리 수정                   | Top-3 중 다른 카테고리 선택       | PATCH 호출 1회, 결과 화면 즉시 갱신                            |
