# Spec. 옷 등록 모바일 UI (item-register-mobile)

> 한 줄 요약. Flutter에서 카메라·갤러리 진입부터 단건/일괄 업로드·진행률·카테고리 수정까지 사용자 흐름을 완성한다.

## 배경

`item-register-api`가 백엔드 엔드포인트(presign·items·batch·PATCH)를 제공하고 `item-register-ai`가 분류 결과를 채워 준다. 본 슬러그는 이를 소비해 사용자가 옷 한 벌을 8초 안에, 갤러리 20장을 백그라운드로 옷장에 넣을 수 있게 한다.

기존 mobile-foundation 슬러그가 라우팅·Riverpod·Dio 클라이언트·Sentry까지 끝낸 상태이므로 본 슬러그는 `features/item_register/` 디렉토리에 흐름·화면·상태·API 어댑터를 추가한다.

## 기능 목록

### F1. 단건 등록 흐름

옷장 화면 "+" 진입 → 카메라/갤러리 선택 → 촬영/선택 → 미리보기 → 저장.

**동작 방식**

- 카메라/저장소 권한 확인. 거부 시 설정 진입 안내 화면.
- 카메라. 가이드 프레임. 촬영 후 재촬영 가능.
- 갤러리. 1장 선택.
- 클라이언트 이미지 압축 (긴 변 1600px·JPEG quality 85).
- presign → PUT → POST /items.
- 결과 화면. AI 카테고리 표시 + 1탭 수정 진입.

**포함 범위**

- 권한 처리(iOS·Android).
- 압축·업로드·실패 재시도(최대 2회).
- 등록 평균 ≤8초 목표(게이트는 item-register-gates).

**제외 범위**

- 옷 상세 수정(브랜드·태그·계절). → 후속 슬러그.

### F2. 일괄 등록 흐름

갤러리 멀티셀렉트 → 최대 20장 → 백그라운드 큐 → 진행률 → 완료 토스트.

**동작 방식**

- 갤러리 picker 멀티 모드. 21장 이상 선택 시 클라이언트 차단.
- 클라이언트가 각 사진 압축 → POST /items/batch로 batchId·N개 presign 받음.
- 백그라운드 큐가 순차 PUT.
- 2초 간격 GET /items/batch/:batchId 폴링으로 진행률 갱신.
- 헤더 영역에 진행 배지("3/20"), 완료 시 토스트.
- 실패 항목은 별도 목록 화면에서 재시도 버튼 노출.

**포함 범위**

- 멀티셀렉트 UI.
- 백그라운드 큐 + 진행률 폴링.
- 실패 목록·재시도.

**제외 범위**

- OS 푸시 알림. → MVP 인앱 토스트만.

### F3. 카테고리 1탭 수정

AI 분류 결과를 사용자가 즉시 수정.

**동작 방식**

- 등록 결과 화면 또는 옷 상세 화면에서 카테고리 칩 옆 "수정" 아이콘.
- 모달. Top-3 큰 버튼 + "더 보기" → 8개 전체 목록.
- 선택 시 PATCH /items/:id/category 호출. 별도 저장 버튼 없음.

**포함 범위**

- Top-3 노출(분류 API 응답을 결과 화면에 들고 와서 사용).
- 8개 전체 목록 폴백.

**제외 범위**

- 색상·태그 수정. → 후속.

### F4. 미분류 임시 노출

옷장 그리드(아직 별도 슬러그)가 완성 전이라도, AI 신뢰도 미달로 미분류 상태인 Item이 등록 결과에서 바로 인지되도록 한다.

**동작 방식**

- 등록 결과 화면에서 category=null이면 "미분류" 라벨 + 즉시 수정 모달 자동 노출.

**포함 범위**

- 등록 직후 자동 분기.

**제외 범위**

- 옷장 화면의 미분류 섹션. → `closet-browse`.

## 입출력

본 슬러그는 백엔드 엔드포인트를 소비만 한다. 사용자 입력은 카메라/갤러리 픽 + 카테고리 선택. 화면 출력은 미리보기·진행률·결과·실패 목록.

## 제약 조건

- 클라이언트 압축 후 단건 평균 0.5~1.5MB.
- 백그라운드 큐는 OS 제약 내(앱 백그라운드 진입 후 3분 정도). 그 이상은 다음 포그라운드 진입 시 재개.
- 위젯 테스트로 권한 거부·압축 실패·업로드 실패 경로 검증.

## 예외 케이스

- 카메라 권한 거부 → 권한 안내 화면 + 설정 진입.
- 저장소 권한 거부(갤러리) → 동일.
- 네트워크 단절 → 로컬 큐 적재, 복귀 시 자동 재시도.
- 사전서명 만료 → 클라이언트 1회 재발급·재시도.
- 21장 초과 선택 → 토스트 + 차단.
- 백그라운드 PUT 중 OS가 앱 종료 → 다음 진입 시 미완 항목 큐 재구성.

## 채택 근거

**핵심 이유**

- 등록 마찰 8초 게이트는 클라이언트 압축·권한 흐름·UI 즉시성이 좌우한다. 백엔드만으로 달성 불가.

**보조 이유**

- 백그라운드 큐로 사용자가 다른 화면 이동 가능 → 옷장 초기 시드 채우기에서 이탈 방지.

**기각된 대안**

- **단건 등록만 MVP에 포함.** 옷장 초기 시드 50벌을 한 장씩 50번 등록하면 마찰 너무 큼.
- **카메라 SDK 직접 구현.** image_picker + camera 패키지로 충분.

## 비기능 요건

**성능**

- 등록 평균(카메라 진입~저장 완료). ≤8초(실기기 3종 평균. 게이트는 item-register-gates).
- 배치 진행률 폴링. 2초 간격.

**보안**

- 권한 안내에서 권한 사용 목적 명시(앱스토어 거부 방지).
- 토큰·objectKey는 SecureStorage 또는 메모리만, 디스크 평문 저장 금지.

**확장성**

- 단일 사용자 동시 배치 1개 가정. 두 번째 요청은 안내 후 차단.

## 용어 정의

- **로컬 큐.** Flutter 측 업로드 작업 대기열. 네트워크 단절·앱 백그라운드 종료 시 작업 재개에 사용.
- **미분류 자동 모달.** 등록 직후 category=null 감지 시 자동으로 카테고리 선택 모달을 띄우는 UX.
