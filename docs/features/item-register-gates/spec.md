# Spec. 옷 등록 게이트 검증 (item-register-gates)

> 한 줄 요약. 옷 등록 슬러그 3종(api·ai·mobile)이 PRD Tigers 게이트(T1·T4)와 NFR 부하 기준을 통과하는지 측정·증명한다.

## 배경

PRD의 출시 게이트 중 옷 등록과 관련된 항목은 다음 3개다.

- **T1 AI 분류 정확도** — 회귀 셋 100장 Top-1 ≥80%, Top-3 ≥95%.
- **T4 등록 평균 시간** — 실기기 3종 평균 ≤8초, `/classify` P95 ≤2초.
- **NFR 부하** — `/classify` 동시 1,000 요청에서 에러율 ≤0.5%.

이 게이트들은 코드 추가가 아닌 **측정·증명·문서화** 작업이다. 각각 모델·실기기·부하 환경이 필요해 본 슬러그로 묶어 출시 직전 일괄 수행한다.

## 기능 목록

### F1. T1 정확도 게이트

회귀 셋 수집·라벨링 + 자동 측정 + 결과 보고.

**동작 방식**

- 카테고리 8종 × 약 12장 = 회귀 셋 100장 수집(공개 데이터셋·자가 촬영 혼합).
- 라벨링 JSON. `{ filename, expectedCategory }`.
- 스크립트가 `POST /classify` 호출, Top-1·Top-3 정확도 계산.
- 임계값 미달 시 게이트 실패. 모델 또는 데이터 재검토.

**산출**

- `docs/features/item-register-gates/regression-set/` 디렉토리(라벨 JSON·이미지는 외부 스토리지 참조).
- 측정 보고 `docs/features/item-register-gates/reports/t1-{date}.md`.

### F2. T4 등록 시간 게이트

실기기 3종 측정 + 자동 인스트루멘트.

**동작 방식**

- 클라이언트에 등록 흐름 타이머 추가(카메라 진입~결과 화면). Sentry 또는 분석 이벤트로 전송.
- 측정 기기. iOS 1종(아이폰 13 표준급), Android 저사양(갤럭시 A 시리즈), Android 고사양(갤럭시 S 시리즈).
- 각 기기 10회 측정. 평균·P50·P95 계산.
- 평균 ≤8초 통과.

**산출**

- 측정 보고 `docs/features/item-register-gates/reports/t4-{date}.md`.

### F3. /classify 부하 게이트

분류 서비스 동시 1,000 요청 부하.

**동작 방식**

- k6 또는 Locust로 동시 1,000 vu, 회귀 셋 100장 반복 요청.
- P95 ≤2초 + 에러율 ≤0.5% 통과 기준.

**산출**

- 측정 보고 `docs/features/item-register-gates/reports/classify-load-{date}.md`.

### F4. 분석 이벤트 결선 검증

`item_registered`·`classify_corrected`·`batch_failed` 이벤트가 실제 분석 파이프라인에 정상 도착하는지 확인.

**동작 방식**

- 등록 1회·수정 1회·배치 실패 1회 인위 발생.
- AnalyticsService 콘솔 sink 또는 Amplitude/Mixpanel 콘솔에서 수신 확인.

**산출**

- 체크리스트 `docs/features/item-register-gates/reports/analytics-{date}.md`.

## 입출력

본 슬러그는 코드 추가가 거의 없고 측정 도구·스크립트·보고서가 산출물이다.

## 제약 조건

- 실기기 3종 보유 필요.
- k6/Locust 환경(로컬 또는 CI 컨테이너).
- 회귀 셋 사진은 저작권·초상권 문제 없는 것만(공개 데이터셋 또는 자가 촬영).

## 예외 케이스

- T1 임계값 미달 → 모델 재선정(`item-register-ai` 로 회귀).
- T4 미달 → 클라이언트 압축·업로드 병렬화·UI 즉시성 보강(`item-register-mobile` 로 회귀).
- 부하 미달 → μsvc 수평 확장 또는 모델 경량화.

## 채택 근거

**핵심 이유**

- PRD Tigers는 출시 가능 여부를 결정하는 객관 측정 항목이다. 측정 없이 "구현 완료"로 끝낼 수 없다.

**보조 이유**

- 게이트를 별도 슬러그로 분리하면 어느 슬러그가 회귀를 일으켰는지 격리하기 쉽다.

**기각된 대안**

- **각 슬러그 DoD에 게이트 측정을 넣는다.** 3개 슬러그가 동시에 끝나야 측정이 가능해 결합이 강해짐. 별도 슬러그로 두는 게 단순.

## 비기능 요건

본 슬러그가 다른 슬러그의 NFR을 측정하므로 별도 NFR 없음.

## 용어 정의

- **회귀 셋.** 모델 변경 시마다 동일 입력으로 측정해 회귀 여부를 판정하는 라벨링된 사진 세트.
- **인스트루먼트.** 등록 흐름 단계별 시간을 측정하기 위해 클라이언트에 삽입하는 타이머·이벤트.
