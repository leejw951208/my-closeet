# OKRs. My Closet

- **Date.** 2026-05-17
- **Cadence.** 분기 단위. MVP 출시 후 첫 분기 = Q-LAUNCH로 표기.
- **관련 문서.** [../founder/north-star.md](../founder/north-star.md), [metrics-dashboard.md](metrics-dashboard.md), [prd-mvp.md](prd-mvp.md), [../marketing/gtm-plan.md](../marketing/gtm-plan.md)

## OKR 작성 원칙

1. Objective는 정성적·영감적. 1문장으로 외울 수 있어야 한다.
2. Key Result는 정량적·이진적. NSM 또는 Input Metric에서 직접 파생한다.
3. KR은 분기당 3~5개. 7개 넘으면 우선순위가 없는 것과 같다.
4. KR 달성률 70%가 목표(stretch). 100% 달성이면 너무 보수적인 목표였다는 신호다.

## Q-PREP (개발·검증 분기, MVP 출시 전 12주)

### O1. 출시할 수 있는 진짜 가치를 가진 MVP를 완성한다.

- KR1. AI 카테고리 분류 Top-1 정확도 ≥80% 회귀 셋에서 달성. (현재 0)
- KR2. 신규 가입자 7일 누적 등록 옷 ≥20벌 (클로즈드 베타 30명 평균). (현재 N/A) — _5벌 챌린지 완주율 KR을 대체. 강제 챌린지 없이도 등록 임계를 넘기는지가 진짜 질문._
- KR3. 사진 등록 평균 소요 시간 ≤8초 (실기기 3종 평균). (현재 N/A)
- KR4. 페이퍼 프로토 코디 보드 만족도 ≥7/10 (8명 기준). (현재 N/A)
- KR5. PRD P0 스토리 모두 코드 머지 + QA 통과. (v0.2 기준 US-05·US-11 제외)

### O2. 출시 첫 주의 실패 가능성을 차단한다.

- KR1. Pre-mortem Launch-Blocking Tigers 4건(T1·T3·T4·T5) 모두 mitigation 완료. (T2는 v0.2에서 해제.)

- KR3. 법무 자문 통과 + 약관·개인정보·14세 차단 플로우 검증.
- KR4. 분석 SDK + KPI 대시보드 5종 가동 (PRD 출시 전 체크리스트 100%).

## OKR 운영 룰

- **주간.** Daily Pulse + Weekly Review에서 KR 진척 색상 표기 (Green ≥70%, Yellow 40~70%, Red <40%).
- **월간.** OKR 1:1 회고. 미달 사유 + 다음 달 액션 정리.
- **분기 종료.** 최종 점수 + 회고 (`pm-execution:retro`). 70% 목표 기준 평가.
- **변경 룰.** KR 목표값은 분기 중 변경 불가. Objective는 시장 변동 시 회고 후 변경 가능.
- **연계.** 모든 KR은 [metrics-dashboard.md](metrics-dashboard.md)의 측정 가능 지표여야 한다. 측정 불가능한 KR은 만들지 않는다.
