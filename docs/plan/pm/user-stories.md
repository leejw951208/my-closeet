## Epic A. 계정·온보딩

### A1. 소셜 로그인 (카카오·애플·이메일)

**As a** 신규 사용자, **I want** 카카오 또는 애플 계정으로 로그인하고 싶다, **so that** 별도 가입 절차 없이 빠르게 시작할 수 있다.

- [ ] 카카오·애플·이메일 3가지 로그인 버튼이 첫 화면에 노출됩니다.
- [ ] 카카오 로그인 시 OAuth 동의 후 자동 계정 생성됩니다.
- [ ] 신규 가입자는 약관·개인정보·14세 이상 확인 동의 체크를 한 화면에서 받습니다.
- [ ] 가입 완료 → 온보딩 첫 화면까지 평균 30초 이내(QA 측정).
- [ ] 로그아웃·회원 탈퇴 기능 포함.

Priority. **P0** | Effort. **M** | Dependencies. 없음.

### A2. 5벌 등록 챌린지 온보딩 — **MVP 제외, v1.5 재기획**

**As a** 신규 사용자, **I want** 가입 직후 5벌 등록 챌린지를 진행하고 싶다, **so that** 첫 코디 발견까지 빠르게 도달할 수 있다.

> **2026-05-17 결정.** 이 스토리는 MVP에서 제외하고 v1.5로 이연합니다. 강제 챌린지가 P1~P4 페르소나 모두에 동일하게 가치를 주는지 미검증이고, 완주 시 "혜택" 카피·인센티브가 비어 있어 베타에서 완주율을 추정하기 어렵습니다. 재기획 시 (1) 챌린지 vs 자유 등록 A/B, (2) 구체 혜택 정의, (3) 임계 5벌/10벌 결정이 선결 조건입니다. MVP에서는 자유 등록 + 등록 N벌 도달 시 첫 코디 자동 제안 토스트로 대체합니다.

- [ ] 가입 직후 챌린지 인트로 화면(목표·예상 시간·혜택)이 노출됩니다.
- [ ] 진척도가 1/5 ~ 5/5로 시각화됩니다.
- [ ] 5벌 등록 완료 시 자동으로 첫 코디 1개가 생성되어 보여집니다.
- [ ] 스킵 가능하되, 스킵 시 홈에서 "5벌 챌린지 이어하기" 진입점을 항상 노출합니다.
- [ ] 챌린지 시작·중단·완료 이벤트가 로깅됩니다.

Priority. **Deferred (v1.5)** | Effort. **M** | Dependencies. B1, C1.

## Epic B. 옷 등록

### B1. 단일 사진 등록 (카메라·갤러리)

**As a** 사용자, **I want** 카메라로 옷을 찍거나 갤러리에서 한 장 선택해 등록하고 싶다, **so that** 내 옷을 디지털 옷장에 추가할 수 있다.

- [ ] 등록 버튼 → 카메라/갤러리 선택 시트가 나타납니다.
- [ ] 촬영·선택 후 자동 배경 제거가 적용된 미리보기가 노출됩니다.
- [ ] 사용자가 배경 제거 결과를 "다시" 또는 "원본 사용"으로 1탭 변경 가능.
- [ ] 저장 시 사진은 암호화 업로드되며, 실패 시 재시도 큐에 들어갑니다.
- [ ] 등록 평균 소요 시간(카메라 진입~저장 완료) ≤8초 (QA 측정).

Priority. **P0** | Effort. **L** | Dependencies. C1 (배경 제거 API).

---

### B2. 갤러리 일괄 등록 (최대 20장)

**As a** 사용자, **I want** 갤러리에서 옷 사진 여러 장을 한 번에 선택해 등록하고 싶다, **so that** 옷장 초기 시드를 빠르게 채울 수 있다.

- [ ] 멀티 셀렉트 UI에서 최대 20장 선택 가능.
- [ ] 백그라운드 큐로 순차 처리, 진행률 (X/20) 표시.
- [ ] 처리 중에도 사용자는 다른 화면 이동 가능.
- [ ] 실패 항목은 별도 목록으로 분리하여 재시도 버튼 제공.
- [ ] 처리 완료 시 푸시 알림 또는 인앱 토스트로 알림.

Priority. **P0** | Effort. **L** | Dependencies. B1.

---

### B3. AI 자동 카테고리 분류

**As a** 사용자, **I want** 등록한 옷의 카테고리(상의/하의/원피스/아우터/신발/모자/가방/액세서리)가 자동으로 부여되었으면 한다, **so that** 수동 입력 노동을 줄일 수 있다.

- [ ] 등록 직후 Top-1 카테고리가 제안되어 표시됩니다.
- [ ] Top-1 정확도 ≥80% (E4 PoC 기준, 출시 전 100장 검증).
- [ ] Top-3 후보가 1탭 거리에 있어 빠르게 수정 가능.
- [ ] 분류 실패 시 "미분류"로 들어가며 사용자에게 직접 선택 요청.
- [ ] 사용자 보정 이벤트가 모델 개선용으로 익명 수집 (동의 시).

Priority. **P0** | Effort. **L** | Dependencies. C1.

---

### B4. 옷 상세 편집 (카테고리·색상·태그)

**As a** 사용자, **I want** 등록한 옷의 카테고리·색상·태그를 수정하고 싶다, **so that** 내 옷장 데이터가 정확하게 정리된다.

- [ ] 옷 상세 화면에서 카테고리·색상·태그(자유 텍스트, 5개까지) 편집 가능.
- [ ] 색상은 자동 추출값을 기본값으로 제공.
- [ ] 변경 사항은 자동 저장(별도 저장 버튼 없음).
- [ ] 옷 삭제 시 "휴지통"으로 30일 보관 후 영구 삭제.

Priority. **P0** | Effort. **M** | Dependencies. B1.

## Epic C. AI 인프라

### C1. 배경 제거 + 카테고리 분류 API

**As a** 시스템, **I want** 사진 1장을 받아 배경 제거 결과와 카테고리 Top-3을 반환하는 단일 API를 제공하고 싶다, **so that** 클라이언트가 등록 흐름을 단순하게 호출할 수 있다.

- [ ] POST /classify 엔드포인트, multipart/form-data 사진 입력.
- [ ] 응답. 배경 제거 이미지 URL + 카테고리 Top-3 (라벨 + 신뢰도).
- [ ] P95 응답시간 ≤2초 (3G 환경 제외).
- [ ] 동시 1,000 요청에서 에러율 ≤0.5%.
- [ ] 모델 버전·신뢰도·요청 ID가 로깅됩니다.

Priority. **P0** | Effort. **L** | Dependencies. AI 모델 PoC(E4) GO.

## Epic G. 추천·푸시

### G1. 날씨 기반 일일 코디 추천 (규칙 기반) — **MVP 제외, v1.5 재기획**

**As a** 사용자, **I want** 매일 아침 9시 오늘 날씨에 맞는 코디 추천 푸시를 받고 싶다, **so that** 빠르게 오늘 옷을 결정할 수 있다.

> **2026-05-17 결정.** 이 스토리는 MVP에서 제외하고 v1.5로 이연합니다. 핵심 이슈는 **"언제" 알릴 것인가**입니다. 사용자가 이미 외출한 후 도착하는 푸시는 효용이 0이며, 매일 아침 9시 일괄 발송은 코호트별로 정답이 다릅니다(직장인·학생·프리랜서 출발 시각이 모두 다름). 재기획 시 (1) 시각 트리거 대신 컨텍스트 트리거(기상 알람·캘린더·위치 변화) 활용 가능성, (2) 푸시 없이 인앱 위젯·홈 카드만으로 충분한지, (3) 일괄 푸시의 OFF율 보호 방법을 우선 검토합니다. MVP에서는 셔플(US-09)이 "오늘 뭐 입지" 결정 도구 역할을 합니다.

- [ ] 사용자 위치(권한 허용 시)로 날씨·기온 조회.
- [ ] 온도 구간 × 강수 여부 룰로 카테고리 매핑(예. 8~15℃ → 긴팔+가벼운 아우터).
- [ ] 보유 옷 중 매칭되는 조합 1~2개 생성, 푸시로 발송.
- [ ] 푸시 탭 → 추천 코디 상세, "오늘 입기" 1탭 저장.
- [ ] 위치 권한 거부 시 사용자가 지역(시/도)을 수동 선택.

Priority. **Deferred (v1.5)** | Effort. **L** | Dependencies. E1, F1, 날씨 API, 발송 타이밍 모델(별도 기획).

## Technical Notes (Cross-cutting)

- **API 버전 관리.** v1 prefix 고정 (`/api/v1/`).
- **사진 처리.** 클라이언트에서 리사이즈(최대 1600px) 후 업로드. 원본 보존하지 않음.
- **온/오프라인.** 옷 등록·코디 저장은 오프라인 큐, 백그라운드 동기화.
- **이미지 캐싱.** Disk LRU 200MB, 메모리 LRU 50MB.
- **분석 SDK.** 출시 전 한 곳으로 통일 (Amplitude 또는 Mixpanel).
- **푸시 토큰 관리.** 로그아웃·재로그인 시 토큰 갱신 처리.
- **모델 핫스왑.** AI 모델은 서버 사이드 라우팅, 버전 롤백 가능 구조.
- **테스트 계정.** QA용 시드 계정 + 100벌 더미 데이터 스크립트 (P1-03 위젯 검증용).
