## 1. Executive Summary

20대 SNS 패션러(P1)가 사진 한 장으로 자기 옷장을 빠르게 디지털화하고, 자기 옷으로 새로운 코디 조합을 발견·공유할 수 있는 모바일 앱입니다. MVP는 "등록 마찰 최소화 + 코디 발견의 즐거움"이라는 단일 가치 동사에 집중하며, 출시 후 D7 retention 30%, WAU당 코디 생성 1.5회 달성을 목표로 합니다.

> **v0.2 변경.** "5벌 등록 챌린지(US-05)"와 "날씨 기반 일일 추천 푸시(US-11)"는 MVP에서 제외하고 v1.5로 이연합니다. 챌린지는 강제 온보딩이 페르소나 4종에 일관된 가치인지 더 검증한 뒤 카피·인센티브를 재설계합니다. 추천 푸시는 "**언제** 알릴 것인가"가 콘텐츠보다 중요하다는 판단(이미 외출한 사용자에게는 의미가 없음)이라, 발송 타이밍 모델을 별도 기획 라운드에서 정의한 뒤 도입합니다.

## 2. Background & Context

- **문제 공간.** 옷은 많은데 입을 게 없다는 페인은 보편적이지만, 기존 옷장 앱(Acloset, Whering 등)은 등록 노동이 크고 한국 사용자 맥락에 최적화되어 있지 않습니다. 한국에서는 Acloset이 사실상 단독이며 무료 100벌 벽이 약점입니다.
- **사용자 리서치.** P1~P4 4개 페르소나 중 P1을 비치헤드로 선정. 모든 페르소나가 "등록 30벌 임계 도달 전 이탈"을 공통 위험으로 가집니다. ([../researcher/user-personas.md](../researcher/user-personas.md))
- **경쟁 환경.** 글로벌 리더(Whering, Indyx)는 한국 미진출, 국내 Acloset이 정면 경쟁자. 시장 갭으로 ①등록 마찰 완화, ②중복구매 경고, ③"어제 뭐 입었지" 회상, ④OOTD 1탭 SNS 공유가 비어 있습니다. ([../researcher/competitor-analysis.md](../researcher/competitor-analysis.md))
- **What triggered.** 신규 제품 콘셉트 검증 단계. MVP는 디스커버리 실험(E1~E5)에서 GO 판정이 나오는 것을 전제로 진행합니다.

## 3. Objectives & Success Metrics

### Goals

1. P1(SNS 패션러) 사용자가 첫 7일 내에 20벌 이상 등록하고 첫 코디를 만들 수 있게 합니다.
2. 사용자가 자기 옷장을 한눈에 보고, 마네킹 보드·셔플로 새 조합을 능동적으로 발견할 수 있게 합니다.
3. 사용자가 자기 OOTD를 1탭으로 인스타·틱톡에 공유할 수 있게 합니다.
4. 출시 후 90일 내 D7 retention 30%, WAU당 코디 생성 1.5회를 달성합니다.

### Non-Goals (MVP에서 명시적으로 하지 않는 것)

1. **가족·친구 옷장 공유.** v1.5로 연기. 1인 가치 검증이 선결입니다.
2. **중복구매 가드(쇼핑몰 연동).** v1.5 유료 모듈. P4 타겟 검증 후 진행합니다.
3. **결제·구독 시스템.** MVP는 성장 우선, 전부 무료입니다.
4. **인간 스타일리스트 상담.** Cladwell 영역, 진입하지 않습니다.
5. **남성 패션 특화 콘텐츠·커뮤니티.** v1에서는 비대상입니다.
6. **글로벌 다국어 지원.** 한국어·한국 시장 우선입니다.
7. **웹 버전.** 모바일 First. iOS + Android만.
8. **고급 통계 대시보드(컬러 분석, 활용률 점수 등).** v1.5 P3 타겟 모듈입니다.

## 4. Target Users & Segments

- **Primary. P1 SNS 패션러 (가설 35%).** 20대 여성, 인스타·핀터레스트 OOTD 폴더 보유, 매일 코디 결정에 5~15분 사용. 같은 조합 반복과 영감-내 옷 단절이 핵심 페인. 자세히는 [../researcher/user-personas.md](../researcher/user-personas.md) 참조.
- **Secondary (관찰만, 최적화는 v1.5에서). P2 결정 피로 직장인 (가설 25%).** MVP의 추천·캘린더 기능에서 일부 가치를 받지만 최적화 대상은 아닙니다.
- **Not serving in MVP.** P3(미니멀·정리광), P4(절약·가족관리자), 남성 패션 사용자.

## 6. Solution Overview

### High-level 아키텍처

- **클라이언트.** React Native (또는 Flutter) — iOS + Android 동시. 사진 처리·온디바이스 추론은 네이티브 모듈로 wrap.
- **백엔드.** 가벼운 API 서버 (Node/TS 또는 Python FastAPI) + Postgres + Object Storage(S3 호환).
- **AI 파이프라인.** 배경 제거(예. MODNet/U2Net) + 카테고리 분류(CLIP 기반 fine-tune 또는 사전학습 fashion classifier). 우선 서버 추론으로 출시, 안정화 후 온디바이스 이전.
- **알림.** FCM 단일 채널.
- **분석.** Amplitude 또는 Mixpanel + Sentry.

### 핵심 디자인 결정

1. **등록 마찰 최소화가 0순위.** 카메라 진입~저장까지 8초가 KPI. 갤러리 일괄 import가 온보딩 핵심. 강제 챌린지(US-05) 대신 자유 등록 + "20벌 채우면 코디 보드가 풍부해진다"는 약한 가이드만 노출.
2. **이모지·사진 토글.** 화면 우상단 단일 토글로 전체 뷰 전환. 페르소나 호환성 확보.
3. **마네킹 코디 보드.** v1은 슬롯 기반 단순 모델(상/하/아우터/신발 4슬롯). 자유 캔버스는 v1.5.
4. **추천은 푸시가 아닌 인앱 기능으로만 MVP 진입.** 셔플(US-09)이 추천의 자리를 채운다. 푸시 기반 추천(US-11)은 발송 타이밍 모델이 정의된 v1.5에서 도입한다.
5. **OOTD 공유 이미지.** 서버에서 생성하지 않고 클라이언트에서 합성. 워터마크·앱 딥링크 포함.
6. **데이터 모델.** User → Closet → Item → Outfit. Item은 사진·카테고리·색상·태그. Outfit은 Item 참조 배열 + 날짜 + 메타.

### 보안·프라이버시

- 옷 사진은 사용자 식별 가능 정보로 간주. 암호화 저장, 본인 접근만.
- AI 분류용 사진 전송 시 임시 토큰 사용, 분류 후 학습용 보관은 명시적 동의 후에만.
- 한국 개인정보보호법 준수, 14세 미만 가입 차단.

## 7-1. Launch-Blocking Risks & Mitigations (Pre-mortem 반영)

### Fast-Follow 대응 계획 (출시 후 1~2 스프린트)
- T6 인플루언서 자생 확산 미발생 → 2주 단위 캠페인 + 재계약 인센티브 (Owner. 마케팅).
- T7 (v0.2에서 보류) MVP에 푸시 추천이 없으므로 출시 직후 푸시 OFF 리스크 자체가 사라짐. v1.5 푸시 도입 시 재평가 (Owner. PM).
- T8 마네킹 슬롯 한계 (원피스·세트업) → v1.1 카테고리 가중 그래프 (Owner. 디자이너).
- T9 OOTD 공유 이미지 약함 → 워터마크·배경 A/B (Owner. 디자이너).
- T10 네이버 로그인 부재 → 가입 이탈 분석 후 추가 검토 (Owner. 모바일).

### Rollback / Safety
- 코디 보드·셔플·SNS 공유는 서버 사이드 피처 플래그로 즉시 OFF 가능. (푸시 추천은 v1.5 도입 시점에 동일 플래그 적용.)
- AI 모델 정확도 회귀 발견 시 직전 버전으로 5분 내 롤백.
- DB 일 단위 스냅샷 30일 보관. 사용자 데이터 익스포트 출시 시점부터 제공.
