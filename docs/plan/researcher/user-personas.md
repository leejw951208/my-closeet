# User Research Report. 옷장 앱 (My Closet)

- **Date.** 2026-05-17
- **Data sources.** 실데이터 없음. 제품 콘셉트 기반 **가설 페르소나**. 검증 전임을 명시.
- **Confidence.** 낮음 (Hypothesis-grade). E1·E3 실험 후 갱신 필요.

## Executive Summary

옷장 앱의 잠재 사용자는 "옷은 많은데 입을 게 없다"는 페인을 공유하지만, 동기와 행동은 4갈래로 갈린다. ①데일리 결정 피로형, ②패션·셀프표현형, ③미니멀·정리광형, ④절약·관리형. **가장 높은 행동 강도와 등록 의지는 ②패션·셀프표현형**(SNS 친화)에서 기대되고, **가장 높은 지불 의향은 ④절약·관리형**(중복구매 방지 ROI 명확). MVP는 ②를 비치헤드로 잡고 ④로 유료화 확장하는 그림이 합리적이다. 핵심 리스크는 "등록 노동"이며, 모든 페르소나가 등록 마찰에서 이탈할 수 있어 시드 단계 UX가 결정적이다.

## Personas

### P1. 지수 (24, 패션·셀프표현형) — "코디 다 짜놨는데 사진 찍기 귀찮은 게 함정"
- **Who.** 대학생/사회 초년생, 인스타·핀터레스트에 OOTD 저장 폴더 보유.
- **Primary JTBD.** 옷장에서 매일 새로운 조합을 발견하고, 잘 나온 코디를 기록·공유하고 싶다.
- **Key pains.** ①같은 조합만 반복. ②코디 영감 저장은 많은데 내 옷에 적용이 안 됨. ③옷이 어디 있는지 잊어버림.
- **Key gains.** "오 이런 조합도 되네" 발견, OOTD 공유, 잘 입는 옷 통계.
- **Behavior.** 사진 등록 적극, 코디 셔플·드래그&드롭 도구 자주 사용, 통계 자랑.
- **Prevalence (가설).** 약 35%.
- **Surprising hypothesis.** 이모지보다 "본인 사진 콜라주"를 더 원할 수 있음.

### P2. 민지 (31, 데일리 결정 피로형) — "출근 전 5분 만에 결정 좀 끝내고 싶어"
- **Who.** 직장인, 오피스 캐주얼, 옷장은 중간 규모(40~80벌).
- **Primary JTBD.** 아침마다 날씨·일정에 맞는 옷을 빠르게 결정하고, 더 이상 고민하고 싶지 않다.
- **Key pains.** ①결정 피로. ②비슷한 옷 반복 착용. ③여행/출장 패킹 스트레스.
- **Key gains.** 자동 추천, "오늘 이거 어때?" 알림, 캘린더 기록.
- **Behavior.** 등록은 한 번 몰아서, 이후 추천·캘린더 위주 소비.
- **Prevalence (가설).** 약 25%.
- **Surprising hypothesis.** 코디 도구보다 "어제 뭐 입었지?" 회상 기능을 더 자주 사용.

### P3. 윤서 (28, 미니멀·정리광형) — "내가 가진 게 정확히 뭐인지부터 알고 싶다"
- **Who.** 미니멀리스트, 캡슐 워드로브 관심, 정리정돈 앱 적극 사용.
- **Primary JTBD.** 옷장을 한눈에 파악하고, 안 입는 옷은 정리·기부·중고판매하고 싶다.
- **Key pains.** ①뭘 가졌는지 모름. ②안 입는 옷 죄책감. ③색상·소재 분포 시각화 부재.
- **Key gains.** 통계 대시보드, 활용률 점수, 정리 추천.
- **Behavior.** 신중한 등록(라벨 꼼꼼), 통계·필터 기능 핵심.
- **Prevalence (가설).** 약 20%.
- **Surprising hypothesis.** 이모지보다 "데이터·표·차트"를 선호. 미니멀 UI 강하게 요구.

### P4. 현우 (35, 절약·관리형) — "샀는데 비슷한 거 또 있어, 진짜 짜증나"
- **Who.** 부모/가장, 가족 옷장 일부 관리, 충동구매 후회 경험.
- **Primary JTBD.** 옷 구매 전에 비슷한 보유 옷을 확인해 중복구매를 막고, 가족 옷도 함께 관리하고 싶다.
- **Key pains.** ①중복 구매. ②아이 옷 사이즈 추적. ③세탁·관리 이력 부재.
- **Key gains.** 위시리스트 충돌 경고, 옷 가격·구매처 기록, 가족 공유.
- **Behavior.** 등록 동기가 명확(돈), 쇼핑 직전 앱 진입.
- **Prevalence (가설).** 약 20%.
- **Surprising hypothesis.** 유일하게 명확한 **유료 의향**을 보일 가능성 (월 4,900원 이상).

## Customer Journey Map

| Stage | Touchpoints | Emotion | Pain Points | Opportunities |
|-------|-------------|---------|-------------|----------------|
| Awareness | 인스타 릴스, 친구 OOTD 공유, 앱스토어 검색 | 호기심 | "또 사진 찍어야 함?" | 짧은 릴스 데모, "1분 만에 옷장 보기" 후크 |
| Consideration | 앱 스토어 페이지, 첫 스크린샷 | 기대·의심 | 등록 수고 vs 이득 불분명 | 이모지 미리보기·샘플 옷장 체험 |
| Onboarding | 첫 5장 사진 등록, 카테고리 자동분류 | 신선·피로 | **30벌 임계 도달 전 이탈 (최대 드롭오프)** | 사진 일괄 import, 게임화, 진척도 시각화 |
| Aha Moment | 코디 셔플 첫 성공, 통계 첫 공개 | 놀라움·뿌듯 | 결과 품질이 약하면 실망 | 5벌만으로도 가능한 미니 코디 챌린지 |
| Active Use | 매일 아침 추천, 캘린더 기록 | 안도·즐거움 | 추천이 반복적이면 식상 | 날씨·일정 연동, 셔플 다양성, OOTD 저장 |
| Expansion | 가족 공유, 위시리스트, 통계 프리미엄 | 효용감 | 결제 저항 | S4 타겟 유료 트리거(중복구매 경고) |
| Advocacy | OOTD 공유, "옷장 자랑" 챌린지 | 자부심 | 공유 매끄럽지 못함 | 1탭 인스타·틱톡 export, 챌린지 해시태그 |

**최대 드롭오프.** Onboarding의 "30벌 임계" 구간. 모든 후속 가치가 여기 종속.
**최대 환희 포인트.** "내가 가진 옷으로 새 조합 발견" 순간(Aha) + "스타일 리포트 공유"(Advocacy).

## Key Insights

1. **등록 노동이 모든 가치의 관문이다.** 4개 페르소나 전부 동일한 이탈 지점을 가진다. → 갤러리 일괄 import·영수증 OCR·"5벌 챌린지" 같은 마찰 완화 장치가 모든 페르소나에 효과.
2. **"이모지 관리"는 P1에게만 강한 후크다.** P2·P3는 효율·데이터, P4는 절약을 본다. → 이모지는 디폴트가 아니라 **모드 선택**으로 가야 모든 페르소나를 잡는다.
3. **유료 의향과 사용 빈도는 일치하지 않는다.** S1은 자주 쓰지만 안 내고, S4는 가끔 쓰지만 낸다. → 무료 사용층(S1)을 GTM 엔진, 유료층(S4)을 매출 엔진으로 분리 설계.
4. **"코디 도구" 가치는 P1에 가장 크고 P3에는 낮다.** 핵심 가치 동사가 페르소나별로 다르다. → MVP는 단일 핵심 가치(코디 발견)를 P1에 맞춰 좁히는 게 안전.
5. **"어제 뭐 입었지" 회상 기능이 의외로 강한 후크일 수 있다.** P2 가설. 캘린더가 단순 부가 기능이 아니라 retention 핵심일 가능성.

## Recommendations

1. **MVP 비치헤드를 P1 (SNS 패션러)로 좁힌다.** 코디 보드 + OOTD 공유 + 이모지 모드를 핵심 가치로.
2. **온보딩 첫 화면을 "5벌만 등록해보세요" 챌린지로 설계.** 임계 도달 전 이탈 방지.
3. **갤러리 일괄 import + 자동 카테고리 분류**를 0순위 기술 투자. (E4 PoC가 결정 지점.)
4. **이모지/사진/통계 뷰를 토글**로 제공해 P2·P3 이탈 방지.
5. **유료화는 v1.5 이후 P4 타겟 "쇼핑 가드" 모듈로 진입.** 중복 경고·예산 추적·가족 공유 패키지.
6. **캘린더(어제 뭐 입었지) 기능을 MVP에 포함**해 P2 retention 가설을 동시에 검증.

## Open Questions

- 실제 1차 사용자가 P1·P2 중 누구인가? → E1 컨시어지 응모자 분포로 확인.
- "30벌 임계" 가설이 맞는가, 더 낮은가? → E1에서 등록 수 대비 코디 시도 횟수 관찰.
- 이모지 vs 사진 vs 통계 뷰 선호도? → E3 페이퍼 프로토에 3가지 변형 포함.
- P4의 유료 의향은 실제 가격(월 4,900/9,900) 어디서 꺾이는가? → E5 후반 가격 민감도 인터뷰.
- 가족·공유 옷장은 별도 페르소나(P5)로 분리할 가치가 있는가? → 추후 정성 인터뷰.
