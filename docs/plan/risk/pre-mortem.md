# Pre-Mortem. My Closet MVP

- **Date.** 2026-05-17
- **Status.** Draft v0.2 (2026-05-17 — 5벌 챌린지·날씨 푸시 MVP 제외 반영)
- **Scope.** MVP 정식 출시 후 90일 시점에 "실패했다"고 가정하고 원인을 역추적합니다.
- **관련 문서.** [../pm/prd-mvp.md](../pm/prd-mvp.md), [../pm/user-stories.md](../pm/user-stories.md), [test-scenarios.md](test-scenarios.md), [../researcher/competitor-analysis.md](../researcher/competitor-analysis.md), [../founder/startup-canvas.md](../founder/startup-canvas.md)

> **상상 시나리오.** 90일 후. WAU당 코디 생성 0.4회, D7 retention 12%, 신규 가입자의 7일 등록 평균 6벌. 인플루언서 시드는 1회 게시 후 잠잠. Acloset에서 유사 기능 발표. 무엇이 잘못된 것일까요.

## Risk Summary

- **Tigers.** 12 (Launch-blocking 4, Fast-follow 4, Track 3, Deferred 1). T2(5벌 챌린지)는 챌린지 자체가 MVP에서 빠지면서 게이트에서 해제. T7(푸시 OFF)은 푸시 추천이 v1.5로 이연되어 MVP에선 비활성.
- **Paper Tigers.** 4.
- **Elephants.** 4.

## Rollback / Safety Plan

- **피처 플래그.** 코디 보드·셔플·SNS 공유를 서버 사이드 플래그로 즉시 OFF 가능. (푸시 추천은 v1.5 도입 시점에 동일 플래그 적용.)
- **AI 모델 핫스왑.** 정확도 회귀 발견 시 직전 버전으로 5분 내 롤백.
- **DB 백업.** 일 단위 스냅샷 30일 보관.
- **사용자 데이터 익스포트.** 출시 시점부터 "내 옷장 내보내기" 제공 (신뢰 자산).
- **장애 커뮤니케이션.** 인앱 배너 + 트위터·인스타 공식 계정 동시 공지 템플릿 사전 작성.

## Recommendation

**조건부 GO.** Launch-Blocking Tigers 4건(T1·T3·T4·T5) 모두 출시 -2주 전까지 클리어해야 합니다. 특히 **T1(AI 정확도)** 는 데이터 게이트로 못박고, 미달 시 출시 일정을 연기하는 것이 합리적입니다. T2(5벌 챌린지 완주율)는 v0.2에서 챌린지를 MVP에서 제외함에 따라 게이트에서 해제되었습니다.

가장 신경 써야 할 것은 Tigers가 아닌 **Elephants E1·E2**입니다. "Acloset과의 진짜 차별점 하나"와 "팀 사이즈 대비 P0 현실성"을 출시 전에 솔직하게 다시 점검할 것을 권장합니다.
