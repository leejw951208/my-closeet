# North Star Framework. My Closet

- **Date.** 2026-05-17
- **Business Game.** **Attention** (소비자 모바일 앱, MVP 무료 → v1.5+ 구독·광고·제휴 매출).
- **관련 문서.** [startup-canvas.md](startup-canvas.md), [pricing.md](pricing.md), [../pm/prd-mvp.md](../pm/prd-mvp.md), [../pm/metrics-dashboard.md](../pm/metrics-dashboard.md), [../marketing/gtm-plan.md](../marketing/gtm-plan.md)

## 1. 게임 분류

옷장 앱은 사용자의 **시간과 매일의 의사결정 순간**을 차지하는 Attention 게임입니다. MVP는 무료이며, 매출은 사용자가 앱을 매일 열고 가치를 누적할수록 가능해지는 구조(v1.5 구독, v2 광고, v2.5 커머스 제휴)입니다. 따라서 북극성은 "사용자가 핵심 가치를 받은 빈도"를 측정해야 합니다.

## 2. 북극성 후보 비교

| 후보 | 설명 | 강점 | 약점 |
|------|------|------|------|
| C1. **WAU당 코디 생성 수** | 주간 활성 사용자가 만든 코디 평균 개수 | 핵심 가치 동사(코디 발견) 직접 측정 | 단순 셔플 남발로 과대 측정 위험 |
| C2. **"착장 기록" 사용자 수** | 캘린더에 "오늘 입기"를 1회 이상 기록한 주간 사용자 | 실제 옷을 입었다는 행동 시그널, 게이밍 어려움 | 활성 정도(빈도) 측정 어려움 |
| C3. **WAU당 의미 있는 코디 사용 수** | 주간 활성 사용자가 (저장 + 캘린더 기록 + OOTD 공유) 중 1회 이상 한 코디 평균 수 | 가치 전달의 종합 시그널, 게이밍 방지 | 정의가 복잡, 팀 이해도 약함 |

## 3. 추천 북극성

### Metric. **WAU당 "Active Outfit" 수**

> **Active Outfit.** 다음 셋 중 최소 1개를 만족하는 코디. ① 저장되어 24시간 이상 보존됨, ② 캘린더에 "오늘 입기"로 기록됨, ③ OOTD로 공유됨.

- **Definition.** `Σ(Active Outfits in week W) / WAU(W)`
- **Current value.** 0 (출시 전).
- **Target.**
  - 30일 후. ≥1.0 / WAU
  - 90일 후. ≥1.5 / WAU
  - 12개월 후. ≥2.5 / WAU
- **Why this metric.**
  - 단순 "코디 생성"이 아니라 **저장·기록·공유 중 하나의 진짜 행동**을 요구해 게이밍을 방지합니다.
  - WAU 분모를 사용해 "활성 사용자가 실제로 가치를 받았는가"를 측정합니다.
  - 매출 성장의 선행 지표입니다. Active Outfit이 누적될수록 retention과 switching cost(데이터 누적)가 함께 상승하고, 이는 곧 유료 전환과 제휴 매출의 토대가 됩니다.

## 4. 7-Criteria Validation

| Criterion | Pass | Notes |
|-----------|:----:|-------|
| Expresses value delivered | ✅ | 코디 발견은 사용자가 우리 앱을 쓰는 핵심 이유입니다. |
| Leading indicator of revenue | ✅ | Active Outfit 누적 → retention 상승 → MAU·구독 의향 상승 → 매출. |
| Measurable & trackable | ✅ | 단일 이벤트 조합(save + diary + share)으로 정의 가능. |
| Understandable by team | ✅ | "사용자가 만든 진짜 코디 개수"로 1문장 설명 가능. |
| Actionable | ✅ | UX·추천·푸시·온보딩 모든 팀이 영향 줄 수 있는 레버 보유. |
| Not vanity | ✅ | 다운로드·가입 수 같은 표면 지표가 아닌 행동 시그널. |
| Not gameable | ✅ | 단순 셔플은 카운트되지 않고, 저장/기록/공유 중 하나의 능동 행동이 필요합니다. |

## 5. Input Metrics

| Input Metric | Drives NSM By | Owner | Current | 30d / 90d Target |
|--------------|---------------|-------|:-------:|:----------------:|
| I1. 신규 가입자의 7일 등록 옷 수 (평균) | 등록이 많을수록 코디 가능성·다양성 증가 → Active Outfit ↑ | PM + 디자이너 | 0 | 15 / 20 |
| I2. WAU 중 코디 보드 진입률 | 진입한 사용자만 Active Outfit 생성 가능 | 디자이너 + 모바일 | 0 | ≥40% / ≥50% |
| I3. 코디 → 저장 전환율 | 셔플·완성한 코디 중 실제 "저장"으로 이어진 비율 | PM + 디자이너 | 0 | ≥25% / ≥35% |
| I4. 코디 → 캘린더 기록 비율 | "오늘 입기" 클릭 비율 | PM + 디자이너 | 0 | ≥10% / ≥20% |
| I5. 코디 → OOTD 공유 비율 | 공유 버튼 클릭 비율 | 디자이너 + 마케팅 | 0 | ≥8% / ≥15% |
| I6. D7 retention | 가치를 받은 사용자만 다음 주 돌아옴 → WAU 자체 확대 | 전 팀 | — | ≥25% / ≥30% |

## 7. Counter-Metrics (Goodhart 방지)

| Counter-Metric | Protects Against | Threshold (Alert) |
|----------------|------------------|-------------------|
| 등록 후 7일 내 삭제·휴지통 행 비율 | 등록 수 부풀리기 (등록만 하고 의미 없는 옷) | ≥15% 시 경보 |
| 코디당 평균 옷 슬롯 채움 비율 | 1슬롯만 채운 "껍데기 코디" 양산 | ≤60% 시 경보 |
| 캘린더 기록 중 7일 내 다른 코디로 덮어쓰기 비율 | 보상 노린 무의미한 기록 | ≥40% 시 경보 |
| 공유 후 1시간 내 코디 삭제 비율 | 공유 카운트만 노린 더미 | ≥10% 시 경보 |
| (v1.5에서 활성) 푸시 알림 OFF율 | 추천 푸시 남용에 따른 사용자 피로 | ≥30% 시 경보 — MVP는 푸시 미출시이므로 추적 안 함 |
| 앱스토어 평점 | 지표 추구로 사용 경험이 손상되는 신호 | ≤4.0 시 경보 |

## 8. Anti-Patterns Avoided

| 흔한 지표 | 우리가 채택하지 않은 이유 |
|-----------|--------------------------|
| **DAU / MAU** | 활성 = 가치 전달이 아닙니다. 옷장 앱은 매일 열지 않아도 코디를 잘 활용하면 가치가 있는 사례가 있습니다. 활성도만 보면 "푸시 폭격" 같은 잘못된 인센티브가 생깁니다. |
| **누적 다운로드** | 허영 지표. 다운로드 후 첫 5벌도 등록 안 한 사용자는 매출과 무관합니다. |
| **WAU/MAU 비율 (stickiness)** | 옷장 앱 사용 빈도는 자연적으로 매일이 아닙니다. 의류 도메인의 비율(20~40%)을 매출 게임처럼 해석하면 잘못된 결정을 낳습니다. |
| **누적 등록 옷 수** | 등록 수만 추구하면 카탈로그형 사용 패턴이 강화되고, 진짜 가치(코디 발견)는 측정되지 않습니다. |
| **세션 수 / 세션 시간** | Attention 게임이지만, 우리는 "결정 피로를 줄여주는" 가치가 핵심이므로 짧은 세션이 오히려 좋을 수 있습니다. |
| **MRR (매출)** | MVP는 무료, 후행 지표. 출시 90일 의사결정에 무용. |

## 9. 활용 가이드

- **주간 리뷰.** 매주 월요일 NSM + I1~I6 변화 점검, 카운터 지표 알람 확인.
- **OKR 매핑.** I1~I6 중 분기별 가장 약한 2개를 OKR Key Result로 채택.
- **실험 디자인.** 모든 A/B 실험은 1차 가설을 NSM 또는 직접 연결된 Input Metric에 매핑합니다.
- **재검토 시점.** 출시 후 90일 시점에 NSM 정의와 임계값 재검토. 비즈니스 모델이 v1.5 구독으로 전환되는 시점에 또 한 번 재검토합니다.
